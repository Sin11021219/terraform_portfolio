### 【構成の概要】
Udemyの演習を通じてTerraformによる実践的なIaCを習得し、その成果としてCloudFront、ALB、EC2、RDSを組み合わせた高可用性なWeb3層アーキテクチャを構築。

### 【工夫した点】
・パブリックサブネットへのALB・EC2配置と、プライベートサブネットによるRDSの隔離
(NAT Gatewayのコストを削減するためEC2をパブリックサブネットへ配置し、EC2のインバウンドSSHを完全に閉じてSession Managerによる制御とALBからの通信のみにSGを絞り込むことで、パブリックサブネットにありながらも実質的にプライベートと同等の安全性を担保する設計とした点。)


### 【苦労した点】
・最小特権原則に基づく多層防御の徹底とSGの依存関係整理
(ALB ➔ EC2 ➔ RDS 間の通信フローを必要最低限のポートとソースに絞り、コード上でのセキュリティグループの依存関係の整理)

ALB: インバウンドを「CloudFrontからの特定ポート（80/443）」のみ、アウトバウンドを「EC2用SG宛(3000)」に制限。

EC2: インバウンドを「ALB用SGからのアプリポート（80/443）」のみ、アウトバウンドを「RDS用SG宛(3306)」および「S3/Session Manager等のエンドポイント」限定。　　

RDS: インバウンドを「EC2用SGからのDBポート(3306)」のみに絞り、アウトバウンドは原則完全閉塞。
